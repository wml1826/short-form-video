## 面试对练：fps / Hz 细节澄清
- 25fps 两来源：
  - 播放/解码侧：`videoplayer.cpp:380` 用 `av_q2d(video_st->r_frame_rate)` 从文件流元数据动态读取，非写死。
  - 录制/编码/直播/画中画侧：`common.h:28 #define FRAME_RATE (25)` 写死，用于 time_base {1,25} 与 QTimer(1000/25=40ms)。
- 1024/44100≈23.2ms = 单帧 AAC 音频时长（AAC 固定每帧 1024 采样点 ÷ 采样率 44100Hz），是音频编码包的时间长度，不是定时器间隔。
- 纠正误解：SDL 视频定时器间隔 = 1/fps = 40ms（25fps 时），与音频 23.2ms 是两路不同节奏；音频靠 SDL 音频回调（硬件中断）驱动，无定时器。
- videoplayer.cpp:267-268 MAX_AUDIO/VIDEO_SIZE 宏里也用 25 作队列阈值假设。

## 澄清：音频解码由 audio_callback 驱动（非 SDL_AddTimer）
- 注册：`videoplayer.cpp:425-426` wanted_spec.samples=SDL_AUDIO_BUFFER_SIZE(1024)，wanted_spec.callback=audio_callback。
- 触发节奏 ≈ 23.2ms = buffer_samples(1024) / sample_rate(44100)，由声卡硬件晶振驱动（av_sync.md:49 公式 sample_rate/buffer_size 次/秒）。
- 解码触发链：硬件要数据 → audio_callback(videoplayer.cpp:622) → audio_decode_frame 解码一帧 AAC(1024样本)填 stream。
- 与视频区别：有音频时不创建 SDL_AddTimer（仅无音频才用 40ms 软件定时器）；视频线程靠 audio_clock 被带着走。
- 结论：23.2ms 确实出现在音频路径，是 audio_callback 的硬件触发间隔；面试误区是把 audio_callback 与 SDL_AddTimer 都叫"SDL定时回调"。

## 关键修正：两个"1024"别混
- SDL 设备缓冲区：`videoplayer.h:19 #define SDL_AUDIO_BUFFER_SIZE 1024`，`videoplayer.cpp:425 wanted_spec.samples = SDL_AUDIO_BUFFER_SIZE; //采样点`。单位是**采样点(samples)不是字节**。回调间隔 = 1024/44100 ≈ 23.2ms，是后果不是显式设置。
- 内部解码缓冲：`videoplayer.cpp:645 is->audio_buf_size = 1024;`（字节，随即被 audio_data_size 覆盖），用于 audio_buf_index/audio_buf_size 的部分拷贝记账（av_sync.md:304 用 bytes_per_second 换算剩余时长）。与设置回调节奏无关。
- AAC 每帧 1024 采样是巧合，不是回调间隔的来源。
- 字节视角等价：1024采样×2ch×2B(S16)=4096字节，4096/(44100×2×2)=23.2ms。

## 量纲澄清（面试官追问）
- AAC 23.2ms 的量纲：1024 samples(纯计数) / 44100 samples·秒⁻¹ = 秒；"samples"相消得时间。1024 是"一帧含的采样点数"(计数)，非"每秒1024"。与采样率相除合法。
- SDL samples 字段 = 缓冲区**容量**，单位"sample-frame"(采样帧=每声道1个样本成组)，是静态容量非速率。1024 sample-frame 即每声道1024样本。回调节奏=容量/采样率=23.2ms。≠"每秒1024"≠"每视频帧1024"。
- 两处1024都"每声道1024"，故数值巧合相等：AAC每解码一帧产1024样本/声道，SDL每回调消费1024 sample-frame(=1024样本/声道)。

## SDL 在本项目的真实职责（纠正"SDL只用于音频"）
- 视频渲染**不用 SDL**：解码后 YUV→RGB32 包成 QImage，emit SIG_getOneImage 信号 → playerdialog 槽 → MyOpenGLWidget::slot_setImage 用 OpenGL 纹理上GPU渲染(videoplayer.cpp:133-135,211; opengl/myopenglwidget.cpp:80; 面试答案.md:114)。无任何 SDL_CreateWindow/SDL_UpdateYUV 视频显示调用。
- SDL 实际承担：①音频播放(SDL_OpenAudioDevice/audio_callback/SDL_MixAudioFormat)；②视频解码线程(SDL_CreateThread)；③包队列同步(SDL_CreateMutex/Cond/CondWait, PacketQueue.cpp)；④无音频时视频定时调度(SDL_AddTimer, videoplayer.cpp:383)；⑤延时(SDL_Delay)。
- 注意 SDL_Init(SDL_INIT_VIDEO|...) 初始化了VIDEO子系统，但仅初始化≠用于渲染(可能为事件/timer所需)。
- 结论："SDL只用于音频"不准确：本项目 SDL 不参与视频像素渲染(Qt/OpenGL负责)，但负责音频+线程+锁/条件变量+定时器；SDL 仍通过 timer/thread 参与视频调度而非渲染。该误区源于本项目是Qt移植版(原FFmpeg+SDL教程用SDL画视频)。
